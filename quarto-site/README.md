# Quarto backup of benstanley.eu

This folder is a **self-contained, parallel rebuild** of the website that is
currently produced with [**Calepin**](https://github.com/vincentarelbundock/calepin)
(a Typst-native static-site generator) in [`../calepin-site`](../calepin-site).
It rebuilds the **same site with the same functionality using
[Quarto](https://quarto.org/)** instead.

**It is a backup, not the live site.** The online site at
<https://benstanley.eu> is built by `../calepin-site/build.sh` into
[`../docs`](../docs) (GitHub Pages). This Quarto version renders only into its own
`./_site` and **never touches `../docs`**. Keep it here so that, if you ever want
to move back to a Quarto toolchain, you can: build this folder, point GitHub
Pages at its output, and you have the same site.

Nothing in here reads, changes, or depends on the Calepin site. Image, CV and
teaching-icon assets were **copied** (not moved) from `../calepin-site`. You can
delete this whole `quarto-site/` folder at any time and the live site is
unaffected.

## The CV is the single source of truth

`cv/cv.yml` holds every publication and research project and drives **three**
outputs from that one file (exactly as in the Calepin site):

1. **`cv/cv.pdf`** — the downloadable CV (built by `typst compile cv/cv.typ`).
2. **`pages/publications.qmd`** — an R chunk reads `cv/cv.yml` with the `yaml`
   package and renders the `monographs`, `journal_articles`, `book_chapters` and
   `datasets` sections in CV order, with the CV's titles. Entries use the CV's
   markup (`*…*` → bold); `doi:…` and bare URLs become clickable links.
3. **`pages/research-projects.qmd`** — reads `cv/cv.yml` and renders
   `projects_current` and `projects_previous` as the CV's timeline layout.

So: **edit a publication or project once in `cv/cv.yml`, run `./build.sh`, and the
CV PDF and both pages update together.**

## Prerequisites

All already installed on this machine:

1. **Quarto** (`quarto` on `PATH`) — `quarto --version` ≥ 1.5.
2. **R** (`Rscript` on `PATH`) with `yaml`, `rvest`, `httr`. `yaml` powers the
   CV-driven pages; `rvest`/`httr` power the live Media scrape. If `rvest`/`httr`
   are missing, the Media page falls back to a cached snapshot baked into
   `pages/media.qmd` and the build still succeeds.
3. **Typst** (`typst` on `PATH`) ≥ 0.13 — builds `cv/cv.pdf` from `cv/cv.yml`
   via the academicv template (downloaded from Typst Universe on first compile).
4. **Python 3** — runs `scripts/sync-teaching.py`.

## Build & preview

Run from inside this folder:

```sh
cd quarto-site
./build.sh          # builds cv/cv.pdf, regenerates Teaching pages, renders to _site
quarto preview      # live preview (or just open _site/index.html)
```

`./build.sh` is the command to run after editing `cv/cv.yml` (or any page) — it
keeps the CV PDF and the CV-driven pages in sync, refreshes the Teaching pages,
and re-runs the live Kultura Liberalna scrape.

## Notes / deliberate differences from the Calepin site

- **Same look, idiomatic Quarto.** Jost everywhere, dark-red (`#8B0000`) navbar
  and footer, red links with a pink accent — set in `assets/styles/website_style.scss`.
  Page-specific layouts (research-projects timeline, Teaching cards, the
  Pooling-the-Poles embed) are in `assets/styles/styles.css`, ported from the
  Calepin `site.css`. Content pages show a Quarto **title block** (e.g.
  "Publications"); the Calepin site omits page titles. This is the main visible
  difference and is the normal Quarto idiom.
- **Footer** reads "Made with R and Quarto" (the Calepin site says "Made with
  Typst and Calepin"). Same licence line.
- **Live Kultura Liberalna scrape** (`pages/media.qmd`) — a port of the Calepin
  R chunk: scrapes the author page newest-first on every render, wrapped in
  `tryCatch` with a cached fallback list, so the build never fails. The
  `recent_media.lua` filter from the *old* Quarto site is not used (the Calepin
  site didn't port it either).
- **Teaching** (`scripts/sync-teaching.py`) is the Quarto twin of the Calepin
  script: identical parsing/metadata logic, but it emits `.qmd` (raw-HTML
  landing cards + per-course link lists) instead of `.typ`. Links point at the
  rendered artifacts on the Netlify teaching site.
- **No deploy artefacts.** Unlike `../calepin-site/build.sh`, this build does not
  write a `CNAME` or `.nojekyll`, because it is a backup that doesn't deploy.
  Add them (and repoint GitHub Pages at `quarto-site/_site`) if you ever switch
  back to Quarto for the live site.
