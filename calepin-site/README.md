# Calepin replica of benstanley.eu

This folder is a **self-contained, parallel rebuild** of the website that
normally lives in the parent directory, but produced with
[**Calepin**](https://github.com/vincentarelbundock/calepin) (a Typst-native
static-site generator) instead of Quarto.

It is fully isolated: nothing in here reads, changes, or depends on the Quarto
project one level up. The Quarto site (`_quarto.yml`, `index.qmd`, `pages/`,
`includes/`, `R/`, `docs/`, …) is left exactly as it was. Image and CV assets
were **copied** (not moved) into [`assets/`](assets/).

You can delete this whole `calepin-site/` folder at any time and the original
site is unaffected.

## What's here

```
calepin-site/
├── calepin.toml          # site config: title, theme, navbar, social links, footer
├── build.sh              # builds the CV PDF then the site (use this to rebuild)
├── index.typ             # home page (portrait + bio, trestles-style)
├── 404.typ               # not-found page
├── assets/
│   ├── site.css          # brand overlay: dark-red navbar + footer, Minion 3, links
│   ├── profile.jpg       # copied from ../assets/images/profile.jpg
│   └── good-change.png   # copied from ../assets/images/good-change.png
├── cv/                   # copied from ../CV/ — the single source of truth (see below)
│   ├── cv.yml            # ALL CV data: publications, research projects, etc.
│   ├── cv.typ            # academicv document; builds cv/cv.pdf from cv.yml
│   ├── template.typ      # (unused leftover from the academicv template)
│   └── cv.pdf            # GENERATED by build.sh (git-ignored)
├── pages/
│   ├── publications.typ      # reads cv/cv.yml → Monographs/Journal/Chapters/Datasets
│   ├── research-projects.typ # reads cv/cv.yml → Current/Previous research projects
│   ├── media.typ            # LIVE: Kultura Liberalna scraped at build; Polityka.pl / Other static
│   ├── pooling-the-poles.typ # embeds the live Shiny polling app
│   └── good-change.typ       # two-column book page
├── themes/
│   └── site-theme/       # local copy of the built-in `academic` theme + a footer partial
└── README.md
```

The original Quarto site maps to this almost one-to-one:

| Quarto source                        | Calepin source                  |
| ------------------------------------ | ------------------------------- |
| `index.qmd` (about/trestles)         | `index.typ`                     |
| `pages/publications.qmd` + `includes/*` | `pages/publications.typ` (from `cv/cv.yml`) |
| `pages/research-projects.qmd`        | `pages/research-projects.typ` (from `cv/cv.yml`) |
| `pages/media.qmd` + `includes/kultura-liberalna-articles.qmd` | `pages/media.typ` |
| `pages/pooling-the-poles.qmd`        | `pages/pooling-the-poles.typ`   |
| `pages/good-change.qmd`              | `pages/good-change.typ`         |
| `_quarto.yml` navbar/footer/theme    | `calepin.toml` + `assets/site.css` |
| `CV/` (Typst CV)                     | `cv/` (copied; now also feeds the site) |

## The CV is the single source of truth

`cv/` is a copy of the original `../CV/` Typst CV (the original is left untouched).
Its `cv.yml` holds every publication and research project, and it now drives
**three** outputs from that one file:

1. **`cv/cv.pdf`** — the downloadable CV (built by `typst compile cv/cv.typ`,
   the academicv template, unchanged).
2. **`pages/publications.typ`** — reads `cv/cv.yml` with Typst's `yaml()` and
   renders the `monographs`, `journal_articles`, `book_chapters` and `datasets`
   sections, **in the same order and with the same titles as the CV**
   (Monographs first). Entries use the CV's markup (`*…*` → bold); `doi:…` and
   bare URLs are turned into clickable links.
3. **`pages/research-projects.typ`** — reads `cv/cv.yml` and renders the
   `projects_current` and `projects_previous` sections.

So: **edit a publication or project once in `cv/cv.yml`, run `./build.sh`, and the
CV PDF and both site pages update together.** To add/rename/reorder a section,
change it in `cv.yml`; the publications page follows the order of the
publication-type sections there (it picks up keys
`monographs`, `journal_articles`, `book_chapters`, `datasets`).

> The website-only “Recent publications” and “Essays, interviews and reviews”
> sections from the earlier version were removed so the page mirrors the CV
> structure. The Web of Science / Google Scholar citation line is kept as a
> short site-only intro. Say the word if you want “Recent” back (it can be
> derived from `cv.yml` in Typst).

## Prerequisites

1. **Typst ≥ 0.15** — already installed on this machine
   (`typst` at `/opt/homebrew/bin/typst`).
2. **R** (`Rscript` on `PATH`) — needed only for the **Media** page, whose
   Kultura Liberalna list is scraped live at build time and needs the `rvest`
   and `httr` packages (`install.packages(c("rvest","httr"))`). If R is missing
   the Media page falls back to a cached list; the rest of the site (including
   the CV-driven Publications page) builds without R. All present here (R 4.6).
3. **Calepin** — provides the `calepin` binary. Two ways to get it:

   - **The Calepin editor extension (recommended, since you use Positron).**
     Install **“Calepin for Typst”** from Open VSX (search *Calepin* in the
     Positron/VS Code Extensions panel). It bundles the `calepin` binary and adds
     these commands to the Command Palette:
     - **Calepin Typst Compile**
     - **Calepin Typst Watch**
     - **Calepin Typst Stop**
     - **Calepin Typst New**

     Open this `calepin-site/` folder, then run **Calepin Typst Watch** to build
     and live-preview.

   - **Or the CLI installer:**
     ```sh
     curl --proto '=https' --tlsv1.2 -LsSf \
       https://github.com/vincentarelbundock/calepin/releases/latest/download/calepin-installer.sh | sh
     ```
     (Pagefind search is compiled into the binary — no separate install needed.)

## Build & preview

**Use `build.sh`** — it builds the CV PDF first (so the site can include it),
then the site. Run it from inside this folder:

```sh
cd calepin-site
./build.sh                       # builds cv/cv.pdf, then the site into _site/
calepin serve _site --open       # preview the result
```

`./build.sh` is the command to run after editing `cv/cv.yml` (or any page) — it
keeps the CV PDF and the CV-driven pages in sync.

If you only touched non-CV pages, a plain site build also works (it still reads
the existing `cv/cv.yml`, but won't rebuild `cv/cv.pdf`):

```sh
calepin compile calepin-site calepin-site/_site                 # one-off
calepin watch  calepin-site calepin-site/_site --serve --open   # live preview
```

The generated HTML lands in `calepin-site/_site/`. The regenerable `.calepin/`
cache, `_site/`, and `cv/cv.pdf` are all git-ignored (see `.gitignore`).

## Notes, deliberate differences, and things to check

- **Theme.** Uses Calepin's built-in **`academic`** theme (top nav bar + centred
  reading column), which is the closest match to the original horizontal Quarto
  navbar. `assets/site.css` layers the brand on top by overriding the public
  `--calepin-*` tokens: Jost (Google Fonts) throughout, dark-red (`#8B0000`)
  navbar and footer, red links with pink accents. The theme's previous/next
  page-nav was removed from `layouts/webpage.html`.

- **Publications & research projects come from the CV** (`cv/cv.yml`), read with
  Typst's `yaml()` — see “The CV is the single source of truth” above. No R is
  involved on these pages.

- **Live Kultura Liberalna scrape** (port of `R/KL_article_scraper.R`).
  `pages/media.typ` scrapes `kulturaliberalna.pl/autor/ben-stanley` with
  `rvest`/`httr` on every build (newest first), via an R chunk using
  `results: "typst"`. It is wrapped in `tryCatch` with a cached fallback list
  baked into the chunk, so the build never fails if the site is unreachable or
  the packages are missing — it just uses the snapshot. The Polityka.pl and
  “Other” media lists are static; the `recent_media.lua` filter was not ported
  (it only reordered the display).

- **Navbar icons.** Social links use Iconify icon tokens (`{icon:collection:name}`),
  fetched at build time. The mapping mirrors the original
  (academicons for ORCID / Web of Science / Dataverse / Zotero, Font Awesome
  brands for LinkedIn / WhatsApp / Bluesky / GitHub, simple-icons for Google
  Scholar, lucide for e-mail). If any icon fails to download, swap its collection
  in `calepin.toml` — e.g. `{icon:simple-icons:orcid}` instead of
  `{icon:academicons:orcid}`. Browse names at <https://icon-sets.iconify.design/>.

- **Footer.** The built-in `academic` theme has no footer region, so the site
  uses a **local copy of that theme** in `themes/site-theme/` (ejected with
  `calepin new theme … --theme academic`) with one addition: a
  `partials/site-footer.html` partial wired into `layouts/webpage.html` and
  `layouts/landing.html`. The footer renders the `[menus.footer]` entries from
  `calepin.toml` (licence line + “Made with Typst and Calepin”), styled dark-red
  via `assets/site.css`. It is a **sticky footer**: `body` is a flex column with
  `min-height: 100vh` and the content region (`.academic-page`/`.academic-landing`)
  grows, so on short pages (e.g. the home page) the footer sits at the bottom of
  the viewport rather than partway up. To restyle or re-word it, edit those files.

- **Typeface — Jost everywhere.** The whole site is set in **Jost**, loaded from
  Google Fonts via a `<link>` in `themes/site-theme/partials/site-head.html`
  (an `@import` in `site.css` would be ignored — Calepin bundles all CSS into one
  file, so a mid-bundle `@import` is invalid). The `--calepin-font-body` and
  `--calepin-font-heading` tokens in `assets/site.css` both point at Jost. Body
  reading text is `0.92rem`; section headings are `h1 1.25rem / h2 1.08rem /
  h3 0.98rem` (tweak in `assets/site.css`). The CV PDF is also Jost (set in
  `cv/cv.yml`), so print and web match.

- **Navbar width.** The `academic` theme caps the top bar at the narrow reading
  column, which crowded the many links/icons. `assets/site.css` widens the bar to
  the viewport on desktop (`@media (min-width: 48.01rem)`) and keeps each label on
  one line; the ≤48rem mobile dropdown is deliberately left untouched.

- **CV.** The navbar “CV” link points at `cv/cv.pdf`, which `build.sh` generates
  from `cv/cv.yml` via the academicv template (Jost font, unchanged from your
  original CV). It's git-ignored because it's a build artefact. The original
  `../CV/` is left completely untouched.

- **`pdf = false`.** The site builds HTML only (the original has no per-page PDF
  twins, and the Pooling-the-Poles page is an `<iframe>` that has no paged form).
  Flip it to `true` in `calepin.toml` if you want PDF versions of the text pages.
  (Note: with PDF on, the R chunks would also need to render to paged output.)

### Where Calepin finds R

Calepin calls `Rscript` from `PATH` by default. If yours lives elsewhere, point
to it explicitly in `calepin.toml`:

```toml
[executables]
rscript = "/usr/local/bin/Rscript"
```
